Blackout Documentation v2.1.0

Welcome to Blackout - a secure and private educational web environment. This documentation provides detailed information about the features, usage, and capabilities of the Blackout application.

Note: Blackout is designed for educational purposes and to provide a protected browsing experience. It should be used responsibly and in accordance with your institution's policies.

Features

Monitoring Detection

Blackout includes advanced monitoring detection that can identify when your screen is being monitored by software like GoGuardian, Securly, or other classroom management tools. Detection methods include:

  • Pattern recognition for mouse movements to detect automated monitoring
  • Network request monitoring to identify monitoring-related communication
  • DOM element detection to identify monitoring extension elements
  • Focus/blur event analysis to detect screenshots and screen recording
  • Canvas fingerprinting detection
  • WebSocket connection monitoring

Stealth Mode

When monitoring is detected, Blackout can hide content from the monitoring software while displaying a message to you. This allows you to maintain privacy while being monitored.

Stealth mode is enabled by default but can be toggled off if desired. When active, the application will automatically hide content when monitoring is detected.

Bypass Mode

Bypass mode allows you to continue using Blackout even when monitoring is detected. This mode is useful when you need to work on permitted content but still want the monitoring detection capabilities.

Important: When bypass mode is active, monitoring software can see your screen content. Only use bypass mode for appropriate content that complies with your institution's policies.

Bypass mode is enabled by default in version 2.1.0 to provide more flexibility in usage.

Lights-Out Mode New

Type "lights-out" anywhere in the application to activate an enhanced bypass mode that works even when standard detection methods might fail. This feature is useful in environments with aggressive monitoring software.

Warning: Lights-Out mode forces the system to enable bypass mode, which means your screen content will be visible to monitoring software. Use this feature responsibly.

Terminal Interface

The terminal interface provides a command-line experience for interacting with Blackout. Key commands include:

Command Description
help Display available commands and usage information
browse [url] Open a specified URL in the terminal interface
status Display current system status, including monitoring detection
stealth [on|off] Enable or disable stealth mode
bypass [on|off] Enable or disable bypass mode
clear Clear the terminal screen
emergency Activate emergency screen blocking

Usage Tips

Keyboard Shortcuts

Several keyboard shortcuts are available for quick actions:

  • Alt + S - Toggle stealth mode
  • Alt + B - Toggle bypass mode
  • Alt + E - Emergency screen blocking
  • Escape - Clear current input in terminal mode

Best Practices

  • Use bypass mode only for appropriate content that complies with your institution's policies
  • Be aware that typing "lights-out" enables bypass mode, making your screen visible to monitoring software
  • The terminal interface provides the most discreet browsing experience
  • Check the status bar regularly to be aware of your current stealth and monitoring status
  • Use the emergency command or shortcut if you need to quickly hide content
Remember: Blackout is designed to provide a private browsing experience while helping you understand when your screen is being monitored. Use these capabilities responsibly.

Troubleshooting

False Positive Detections

If you experience false positive monitoring detections:

  • Try refreshing the page to reset the detection algorithms
  • Use bypass mode temporarily to continue working without interruptions
  • Check if other browser extensions might be triggering the detection system

Performance Issues

The advanced monitoring detection systems in Blackout may affect performance on older devices. If you experience performance issues:

  • Use the basic editor interface instead of the terminal for better performance
  • Close other browser tabs and applications to free up system resources
  • Consider using bypass mode to reduce the active monitoring checks
